\chapter{VISION DE L'ARCHITECTURE}

\section{Principes}
Toutes les données de la plateforme doivent être organisées autour d'un Data Hub central.

\section{Organisation}
Chaque module possède son propre schéma logique tout en partageant des référentiels communs :
\begin{itemize}
\item utilisateurs
\item entreprises
\item rôles
\item permissions
\item médias
\end{itemize}

\chapter{CHOIX DES TECHNOLOGIES}

\section{Base de données relationnelle}
PostgreSQL (base principale)

\section{Cache}
Redis

\section{Recherche}
OpenSearch ou Elasticsearch (évolution)

\section{Base vectorielle (RAG)}
pgvector sur PostgreSQL ou base vectorielle dédiée

\section{Stockage objet}
Compatible S3 (MinIO en local, cloud en production)
